# 开发视角

关注开发者的日常开发规范、项目管理、代码质量和团队协作。

## 核心原则

1. **约定优于配置**：统一的代码风格和目录结构
2. **接口即合约**：通过 Go interface 定义模块边界
3. **文档即代码**：关键设计决策必须在代码附近有文档
4. **可测试性优先**：每个模块必须可独立测试
