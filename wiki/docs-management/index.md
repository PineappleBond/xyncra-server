# 文档管理视角

关注项目文档的规范、结构和维护流程。

## 主题文档

| 文档 | 说明 |
|------|------|
| [Wiki 规范](wiki-standards.md) | 多视角文档库的规范与约定 |
| [文档编写指南](documentation-guide.md) | 文档写作风格、模板、审批流程 |
| [README 维护](readme-maintenance.md) | 项目主页 README 的维护策略 |

## 核心原则

1. **索引先行**：每层目录必须有 index.md 索引
2. **单主题单文件**：一个文档只讲一个主题
3. **视角分离**：不同读者有不同的入口
4. **持续维护**：文档随代码变更同步更新
